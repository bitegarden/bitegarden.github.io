We’ve just released a new version of **bitegarden Security Assessment for SonarQube&trade; Server**, so you can now measure your code against the latest security standards without leaving your usual SonarQube Server workflow: **OWASP Top 10 2025 and OWASP ASVS 5.0.0**.

<h2>What really changes?</h2>

With this version, you can evaluate your projects’ **compliance against OWASP Top 10 2025**, aligning with the most current threats. **OWASP ASVS 5.0.0 is now included**, featuring a selector that lets you choose the standard version, just like you select branches or time periods in other reports. This update gives you a much more precise view of where you stand and what you’re missing to meet **modern security requirements**.

<h2>What does this plugin do?</h2>

bitegarden Security Assessment for SonarQube™ Server is more than just a **vulnerability viewe**. With this plugin, you can:

- Gather all vulnerabilities detected in SonarQube. <br>

- Map them to recognized standards such as OWASP, CWE, WASC, and CERT. <br>

- Get a detailed, structured, and visual overview of your project’s security posture. <br>

<h2>How do you update to the new version?</h2>

If you’re already using the plugin, follow these steps:

1 - Go to the [Universal Plugin Manager for SonarQube&trade; Server (UPM)](https://marketplace.bitegarden.com/product/bitegardenUniversalPluginManager). <br>

2 - Click update. <br>

3 - And that’s it, you’re now operating with the new standards. <br>

<h2>Haven’t tried it yet?</h2>

You can install the [bitegarden Security Assessment for SonarQube&trade; Server](https://marketplace.bitegarden.com/product/bitegardenSecurity) in your instance and configure it.
